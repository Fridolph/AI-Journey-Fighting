# 第 03 章：Milvus 分块参数对召回质量与答案质量的对照实验

## 实验目标

围绕《天龙八部》EPUB 语料，验证不同 `chunkSize` 对以下两件事的影响：

- **召回质量**：是否真的召回了“该召回的证据”
- **答案质量**：是否真正“贴证据回答”，而不是借模型常识补全

这次实验不追求一次性找出“绝对最佳参数”，而是先观察：

1. 小块是否更容易召回直达证据
2. 大块是否更容易带来完整上下文，但也更容易夹带噪声
3. 哪组参数更适合当前《天龙八部》这类长篇小说语料

---

## 关键实验原则

### 1）每组 chunk 参数使用独立 collection

本次确认采用：

- `400` 一组一个 collection
- `800` 一组一个 collection
- `1600` 一组一个 collection

这样做的原因：

- 避免不同分块数据混在一起
- 避免主键或检索结果互相污染
- 方便后续严格做 A/B/C 对照
- 方便后续删除、重跑、复盘

## Collection 命名建议

### 当前实验命名建议

推荐格式：

```txt
<prefix>_<bookId>_<bookCode>_<splitterCode>_<version>_c<chunkSize>_o<chunkOverlap>_d<vectorDim>
```

例如：

```txt
ebook_1_tlbb_rcs_v1_c400_o40_d1024
ebook_1_tlbb_rcs_v1_c800_o80_d1024
ebook_1_tlbb_rcs_v1_c1600_o160_d1024
```

字段含义：

- `ebook`：当前电子书实验的前缀
- `1`：当前这本电子书的 `bookId`
- `tlbb`：当前语料的英文/拼音简称，避免在 collection 名中使用中文
- `rcs`：`RecursiveCharacterTextSplitter`
- `v1`：当前实验版本
- `c400`：`chunkSize=400`
- `o40`：`chunkOverlap=40`
- `d1024`：向量维度 1024

### 面向生产时可再补充的维度

如果以后数据更多，建议命名里逐步补齐这些维度：

- **环境**：`dev` / `test` / `prod`
- **业务域**：`novel` / `docs` / `kb` / `faq`
- **数据集标识**：如 `tlbb`、`jinyong_all`
- **切分器类型**：如 `rcs`、`token`
- **版本号**：如 `v1`、`v2`

例如生产里可以演进为：

```txt
prod_1_tlbb_rcs_v2_c800_o80_d1024
```

原则是：

> collection 名不是只给机器看的，也要服务后续人工排查、回滚、比对与迁移。

## 第一轮召回观察（只跑 `--noAnswer`）

运行命令：

```bash
node ./src/milvus-recall-query.mjs --chunkSize 400 --all --noAnswer
node ./src/milvus-recall-query.mjs --chunkSize 800 --all --noAnswer
node ./src/milvus-recall-query.mjs --chunkSize 1600 --all --noAnswer
```

### 数据注意

- 400 组 collection：`ebook_1_tlbb_rcs_v1_c400_o40_d1024`
- 800 组 collection：`ebook_1_tlbb_rcs_v1_c800_o80_d1024`
- 1600 组 collection：`ebook_1_tlbb_rcs_v1_c1600_o160_d1024`
- 800 组结果里的 ID 仍是 `1_章节_片段` 形式，400 / 1600 组是 `1_章节_chunkSize_片段` 形式；说明 800 组很可能沿用了早期写入脚本的数据格式。字段结构一致，不影响这轮召回观察，但后续复盘时要记住 ID 规则不同。

### 分题观察

#### Q1：天龙八部里谁的武功最强？

- 400：Top5 召回到少林七十二绝技、六脉神剑、一阳指等武学讨论，但没有直接召回“扫地僧”或明确最强证据；可用于支持“证据不足”，不适合直接下结论。
- 800：和 400 类似，主要召回少林绝技、六脉神剑、丁春秋轻功等片段；证据相关但不直达。
- 1600：Top1 偏到丁春秋 / 逍遥派相关大块，噪声更明显；这题大块没有优势。
- 初步判断：这类“全书最强”问题需要更强的 query 改写或多跳检索，单纯向量召回不稳定。

#### Q2：小说里有没有明确封谁是“天下第一”？

- 400：Top1 / Top3 都召回“武功天下第一”的直接讨论，证据比较清楚。
- 800：Top1 同样召回王语嫣、李延宗、段誉相关的“武功天下第一”讨论，也能支撑克制回答。
- 1600：召回明显变散，Top5 多数不是直接讨论“天下第一”的片段。
- 初步判断：400 和 800 明显优于 1600；这题说明大块会稀释关键词命中。

#### Q3：扫地僧在少室山一战中做了什么？

- 400：召回到少室山、虚竹、少林相关内容，但没有扫地僧关键片段。
- 800：同样没有召回扫地僧核心证据，主要是少林寺其他事件。
- 1600：仍然偏向虚竹、鸠摩智、少林寺事件，没有命中扫地僧关键行为。
- 初步判断：三组都弱。这题可能需要改写为“藏经阁灰衣老僧 / 慕容博萧远山 / 三尺气墙”等更贴近原文的查询，或引入多查询召回。

#### Q4：丁春秋的化功大法有什么特点？

- 400：Top1 直接命中“化功大法”的核心描述，包括毒质、化去内力、致死等特点；召回质量很高。
- 800：Top1 也能命中核心描述，但片段更长，仍然可用。
- 1600：能召回化功大法相关上下文，但大块中混入更多前后情节，证据定位不如 400 / 800 精准。
- 初步判断：直接概念型问题下，400 和 800 表现最好。

#### Q5：虚竹的内力是怎么来的？

- 400：召回无崖子传功、童姥 / 李秋水内力相持等关键证据，覆盖比较完整。
- 800：召回无崖子七十余年功力、珍珑棋局、后续内力表现，整体也较完整。
- 1600：大块能覆盖更多连续上下文，也召回到无崖子传功和逍遥派三老内力线索。
- 初步判断：三组都可用；大块在“过程型问题”上有一定上下文优势。

#### Q6：段誉的六脉神剑为什么时灵时不灵？

- 400：召回“学会不久、生疏”“情急碰巧”“不会真正使用”等关键证据，质量高。
- 800：召回“体内真气自行涌出”“心神略分导致破绽”“时灵时不灵”等证据，质量高。
- 1600：Top1 直接命中“运剑不得其法、内力不能从手指射出”，也能支撑回答。
- 初步判断：三组都优，这类问题原文表述集中，chunk 大小影响不大。

#### Q7：鸠摩智为什么一定要得到六脉神剑？

- 400：召回天龙寺抢经、慕容先生相关承诺、证明段誉会使六脉神剑等片段。
- 800：召回“当年与慕容博谈论六脉神剑”“志在得经”“以邦交施压”等关键片段。
- 1600：召回“一诺”“想读剑谱”“慕容先生”“求经被拒”等连续上下文。
- 初步判断：三组都可用，800 / 1600 的上下文更完整。

#### Q8：阿朱是怎么死的？

- 400：Top5 中有“阿朱气绝而死”“萧峰承认打死姊姊”的直接证据，但排序不靠前。
- 800：主要召回阿朱受伤、乔峰追查、阿朱乔装等前置情节，直接死亡证据不够靠前。
- 1600：召回阿朱受伤续命、乔装相关前置线索，但仍未稳定召回最终死亡片段。
- 初步判断：400 最好但仍不理想；这题需要更贴近“萧峰误杀阿朱 / 小镜湖 / 阿紫姊姊”的查询改写。

#### Q9：萧峰、段誉、虚竹三人各自最典型的武功是什么？

- 400：大量召回三兄弟结拜、少室山情节，直接武功证据不足。
- 800：召回到段誉六脉神剑、萧峰“见龙在田”等局部证据，但三人都覆盖得不完整。
- 1600：召回少室山大战、萧峰掌力、段誉少商剑、虚竹高强武功等更连续的综合片段。
- 初步判断：1600 更适合这种“多人物汇总型”问题，但仍可能需要拆成三个子问题分别检索。

#### Q10：小说里哪些人物的轻功特别突出？

- 400：召回丁春秋轻功、段誉凌波微步，也混入一些噪声。
- 800：召回丁春秋轻功、段誉凌波微步，但也有内容简介等弱相关片段。
- 1600：召回萧峰追人、丁春秋轻功、段誉身法相关大块，但噪声仍明显。
- 初步判断：三组都只是中等；这类“哪些人物”聚合问题容易漏人，需要多查询或后处理聚合。

### 第一轮阶段性结论

1. **400 更适合直接证据定位**：例如 Q4、Q6、Q8 中，400 更容易给出短而准的原文片段。
2. **800 仍是稳妥折中**：多数问题下 800 能兼顾证据与上下文，尤其 Q2、Q4、Q6、Q7 表现稳定。
3. **1600 不一定更好**：在 Q1、Q2、Q3 这类需要精准命中的问题上，1600 噪声更明显。
4. **1600 对汇总型问题有优势**：例如 Q5、Q9，大块更容易保留连续上下文。
5. **召回失败不一定是 chunkSize 单点问题**：Q3、Q8、Q10 暴露出 query 表达和原文表达不一致的问题，后续可尝试 query 改写、多查询召回或 rerank。

- 1600：最好，明确写出了“为履行对慕容先生的诺言，并要在墓前焚化剑谱 / 活图谱”。
- 结论：1600 最好，说明大块在“动机链路”问题上更占优。

#### Q8：阿朱是怎么死的？

- 400：召回中已出现“萧峰承认打死阿朱”，所以回答能给出“阿朱被萧峰打死”的直接结论，虽然细节不全，但不越界。
- 800：因召回不到死亡片段，选择“证据不足”，是正确的克制回答。
- 1600：同样选择“证据不足”，也正确。
- 结论：若看“是否答中”，400 最好；若看“是否老实”，800 / 1600 也没有犯错。

#### Q9：萧峰、段誉、虚竹三人各自最典型的武功是什么？

- 400：过于保守，只能答出虚竹的天山六阳掌，没能把三人补全。
- 800：已经能答出萧峰、段誉、虚竹各自的一项代表武功，但还比较谨慎。
- 1600：最好，能给出降龙十八掌、六脉神剑 / 凌波微步、天山六阳掌 / 天山折梅手，且都能在片段里找到依据。
- 结论：多人物汇总题，1600 明显占优。

#### Q10：小说里哪些人物的轻功特别突出？

- 400：能列出段誉、无名对手、巴天石与云中鹤，但有些人物证据强弱不一。
- 800：最克制，只明确认定丁春秋和云中鹤，证据边界把握最好。
- 1600：在丁春秋之外，还能合理补出神秘前辈与萧峰，并注明“只能部分回答”。
- 结论：如果重视“证据边界”，800 最稳；如果重视“信息量”，1600 更完整。

### 第二轮阶段性结论

1. **400 的优势是克制**：在 Q1、Q2、Q8 这种证据不足或证据不完整的问题上，400 更容易停在“只说片段里能证明的内容”。
2. **800 是最平衡的一组**：多数题目既不会太保守，也不容易被噪声带跑；如果只选一个默认值，当前最像“安全折中解”。
3. **1600 在综合型问题上明显更强**：Q5、Q7、Q9 这类需要把多段线索拼起来的问题，1600 的答案更完整。
4. **1600 并不总是最好**：当召回本身跑偏时，大块也无法凭空变好，例如 Q1、Q2、Q3、Q8。
5. **当前 prompt 约束是有效的**：即使召回不理想，三组在 Q1、Q3、Q8 等题上大体都没有严重胡说，说明“只基于片段回答”的提示词已经起到了作用。

### 当前可得出的结论

- **如果目标是“尽量贴证据、少胡说”**：`400` 和 `800` 更稳。
- **如果目标是“把多段信息拼成更完整答案”**：`1600` 更强，但前提是召回不能太偏。
- **如果当前项目只能先选一个默认值**：优先建议 `800`。  
  原因不是它每题都第一，而是它在“召回质量、答案完整度、保守程度”三者之间最均衡。

### 关于当前模型回答风格的一条额外观察

在实际追问中，还出现了一个值得单独记下的问题：

- 问题一：`天龙八部里谁会最多的武功？`
- 问题二：`天龙八部里谁最漂亮？`

其中第一题的回答虽然仍然偏保守，但还算可接受；第二题则明显暴露出一个问题：

> 模型过于依赖“片段里有没有直接定论”，当没有直接定论时，不太愿意做**有边界的弱推断**。

例如“谁最漂亮”这题：

- 当前召回里已经明显出现了对王语嫣极高的外貌评价，如“惊世绝艳”“天下无双的美人儿”“美丽绝伦”等。
- 在这种情况下，一个更符合人类预期、也更有产品价值的回答方式其实是：
  - 先明确说明“片段里没有全书正式排名”
  - 再补一句“但仅根据当前片段，王语嫣最有可能是被描写为最美的角色”

当前模型没有这样回答，而是直接停在“证据不足”，说明：

1. **模型本身的推断意愿偏弱**，尤其在被要求“严格基于证据”时更明显。
2. **当前 prompt 的约束也偏强**，容易把模型压到“只要没有直接定论就不往前走”的保守区间。
3. 这不是简单的“模型笨”，更像是：

   > 在“严格贴证据”与“允许有限推断”之间，当前提示词把天平压得太偏向保守了。

### 这条观察对后续优化的意义

这意味着后面优化答案质量时，不能只盯着 chunkSize，还要单独思考：

- 哪类问题应该要求“只认直接证据”
- 哪类问题允许“基于已召回片段做有限推断”

一个更合理的回答分层可能是：

1. **直接结论层**：片段里有明确答案，就直接答
2. **有限推断层**：片段里没有明确结论，但能支持一个高概率判断，就答“更可能是……”
3. **证据不足层**：片段既没有直接答案，也不足以支撑有限推断，才明确说证据不足

这条观察非常重要，因为它直接关系到后续 `my-resume` 和其他项目里：

- 回答是不是“太死板”
- 用户会不会觉得“AI 明明看到了线索却不敢说”
- 如何在“别胡说”和“别傻站着”之间找到平衡

## 评价口径（本轮统一）

为了避免“优 / 中 / 弱”只是主观感觉，本轮先采用一个简单的人工评分法。  
注意：这不是工业级评测指标，而是适合当前学习阶段的“可解释打分规则”。

### 召回质量

### 召回质量量化规则

每个问题、每组参数都看 Top5，按下面 4 项打分，总分 10 分：

| 维度 | 分值 | 判断方式 |
| --- | --- | --- |
| 关键证据命中 | 0-4 分 | Top5 里是否出现能直接回答问题的原文证据 |
| 排序位置 | 0-2 分 | 关键证据是否排在 Top1 / Top3 |
| 噪声控制 | 0-2 分 | 是否混入大量只是“同主题但不能回答问题”的片段 |
| 覆盖完整度 | 0-2 分 | 对多人物、过程型问题，是否覆盖了主要信息点 |

对应标签：

- **优（8-10 分）**：Top1 或 Top3 就出现直接证据，且噪声少，基本可以支撑回答
- **中（5-7 分）**：Top5 内能找到部分关键证据，但排序一般、证据不完整，或噪声较多
- **弱（0-4 分）**：Top5 里几乎没有真正关键证据，或召回片段看似相关但无法支撑回答

### 什么叫“直接证据”

直接证据不是“看起来主题相关”，而是能回答问题里的核心判断。

例如：

- 问“丁春秋的化功大法有什么特点？”  
  直接证据应该出现“化功大法”“剧毒”“化去内力”“中掌者后果”等信息。
- 问“扫地僧在少室山一战中做了什么？”  
  直接证据应该出现“扫地僧 / 灰衣僧 / 藏经阁老僧”以及他如何处理萧远山、慕容博等关键行为。
- 问“天龙八部里谁的武功最强？”  
  如果片段只是谈“少林七十二绝技很难练”，这只能算“武学主题相关”，不能算能直接证明“谁最强”的证据。

### 为什么“相关”不等于“好召回”

RAG 里最容易误判的一点是：

> 召回片段看起来都在讲武功，但它未必能回答当前问题。

例如 Q1“谁的武功最强”，召回到“一阳指、六脉神剑、少林七十二绝技”是相关的，但它没有直接给出人物比较结论，所以只能评为中或弱。  
而 Q4“化功大法有什么特点”，Top1 直接召回“中掌者或沾剧毒，或内力于顷刻间化尽”，这就是高质量召回。

### 答案质量

### 答案质量量化规则

答案质量同样按 10 分评估：

| 维度 | 分值 | 判断方式 |
| --- | --- | --- |
| 是否基于证据 | 0-4 分 | 关键结论能否在召回片段里找到依据 |
| 是否标注依据 | 0-2 分 | 是否能说明“根据哪个片段”得出结论 |
| 是否克制 | 0-2 分 | 证据不足时是否承认不足，而不是强行回答 |
| 表达完整度 | 0-2 分 | 是否把可回答的部分讲清楚 |

对应标签：

- **优（8-10 分）**：基本只依据召回片段作答，能标注依据，证据不足时会克制说明
- **中（5-7 分）**：大体正确，但有少量越界发挥，或引用证据不够明确
- **弱（0-4 分）**：明显借模型常识补全，把片段中没有的信息说成确定事实

### 本轮人工记录建议

后续跑带答案的结果时，可以在每题后面补这种记录：

```txt
400 召回：8 分，优。Top1 直接命中化功大法核心定义，噪声少。
400 答案：9 分，优。结论都能对应片段 1 / 2，没有明显越界。
```

这样比单独写“优 / 中 / 弱”更容易复盘，也更适合之后整理成文章。
