# AI 开发小灶 01：从万能设计模板开始

## 这轮目标

现在先不追求一次性把所有 AI 开发套路都吃透。

这一轮只练一个动作：

> 看到需求后，先判断它属于哪类 AI 脚本，再决定要用哪些模块。

先把这个动作练熟，后面再慢慢加结构化输出、记忆、工具调用、RAG、分支这些能力。

---

## 万能设计模板

以后拿到一个 AI 脚本需求，先不要急着写代码。

先写这 5 行：

```text
1. 目标：这个脚本要干嘛？
2. 输入：用户给我什么？
3. 输出：我最后要什么格式？
4. 中间要不要：记忆 / 工具 / 检索 / 分支？
5. 最小链路先怎么跑通？
```

这 5 行的作用是帮自己先定边界。

很多时候不是不会写代码，而是还没想清楚：

```text
这是普通问答？
还是结构化抽取？
要不要 RAG？
要不要工具？
输出是给人看，还是给程序继续处理？
```

边界清楚了，模块自然就好选。

## 第 1 题：总结文章核心观点

### 需求

```text
帮我总结这篇文章的核心观点。
```

### 这是一个什么脚本？

这是一个普通问答脚本，也是总结类脚本。

它的本质是：

```text
文章内容 + 总结要求 -> AI -> 总结文本
```

### 分类

```text
普通问答
```

这里不需要工具调用，也不需要 RAG。

因为文章内容已经直接给了模型，不需要再去外部查资料。

### 最小模块

```text
Prompt + Model + Parser
```

对应到 LangChain 里就是：

```text
PromptTemplate
ChatOpenAI
StringOutputParser
```

### 是否需要增强？

可以加 `Stream`。

原因是总结文章时，回答可能比较长。流式输出不是必须功能，但体验会更好。

---

## 这一题的起手代码

```js
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { PromptTemplate } from '@langchain/core/prompts'
import { StringOutputParser } from '@langchain/core/output_parsers'

const model = new ChatOpenAI({
  model: process.env.MODEL_NAME,
  apiKey: process.env.OPENAI_API_KEY,
  configuration: {
    baseURL: process.env.OPENAI_BASE_URL,
  },
})

const prompt = PromptTemplate.fromTemplate(`
请总结下面文章的核心观点。

要求：
1. 提炼 3-5 个核心观点
2. 每个观点用一句话说明
3. 最后给出一句总结

文章：
{article}
`)

const chain = prompt.pipe(model).pipe(new StringOutputParser())

const result = await chain.invoke({
  article: '这里放文章内容',
})

console.log(result)
```

## 这一题要记住什么？

总结类任务的心智模型：

```text
总结类任务 = 普通问答链
输入重点不是 question，而是 article / content
输出如果只是给人看，用 StringOutputParser 就够了
```

如果以后需求变成：

```text
总结文章，并返回 JSON，包括 title、points、summary
```

那它就不再只是普通字符串输出，而要进入“结构化输出”那一类。

这一轮先记到这里。
