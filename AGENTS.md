# AGENTS 协作与学习规范

## 1. 项目目的（Owner 目标）

本仓库用于记录 AI 学习旅程，核心目标是：

1. 先系统梳理 AI 入门概念（已在 `docs/` 中持续沉淀）。
2. 再进入初级实战阶段：围绕 `examples/` 的 demo 学习 Agent 开发。
3. 将“跑通代码 + 过程理解 + 心得感悟”沉淀为可复用文档。
4. 最终整理为可发布到 VitePress 的文章，服务后续开源分享。

## 2. 当前学习阶段

- 当前已完成部分 AI 入门概念整理。
- 现在进入“结合实战做基础能力”的阶段。
- 学习顺序**不按博主项目顺序**，以 Owner 的个人学习节奏为准。

## 3. 执行流程（必须遵守）

每次学习一个 `examples/` 下的类别时，按以下流程执行：

1. 进入对应 `examples/<category>/`，结合博主文章阅读代码。
2. 在本地把 demo 跑起来，验证关键流程。
3. 在根目录 `drafts/` 下创建同名目录：`drafts/<category>/`（若已存在则复用）。
4. 在该目录新增或更新 Markdown 学习记录，记录：
   - 学习目标与背景
   - 环境准备与运行步骤
   - 关键代码理解
   - 问题与排查过程
   - 心得与下一步
5. 当某一类别学习完成后，在 `drafts/<category>/` 产出一篇阶段总结文章。
6. 最终将成熟内容整理进 VitePress 文档（`docs/`）用于公开输出。

## 4. 文档约定

- 文档语言以**中文为主**，后续不再强制维护双语同步。
- 先沉淀到 `drafts/`，再精选进入 `docs/`。
- 记录强调“可复现 + 可理解 + 可迭代”，避免只贴结论。

## 5. 对协作 Agent 的要求

- 不擅自更改 Owner 学习节奏与目录命名规则。
- 新增学习记录时优先复用现有目录结构，保持一致性。
- README、`docs/` 结构发生方向性变化时，同步更新说明文档。
- 修改尽量小而清晰，不做与当前学习任务无关的大改动。
