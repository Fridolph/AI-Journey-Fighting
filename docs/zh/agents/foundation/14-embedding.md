# 14 · Embedding · 语义坐标之道

---

## 一、核心问题：机器怎么"理解"文字？
机器只认识数字，不认识文字。
经过上一步 Tokenization 得到的分词结果，仍然是文本形式，机器无法直接处理。
所以文字必须先转化成数字——这个过程叫 Embedding（向量化/嵌入）。

$$\text{Token（文本形式）} \rightarrow \text{向量（一组有序数字）}$$

## 二、什么是向量？
向量 = 一个词在高维空间里的坐标。
现代大模型通常使用几百到几千维的高维向量，比如：
- 简单二维示例：`猫 = [0.9, 0.7]`，`狗 = [0.85, 0.75]`，`火箭 = [0.1, 0.2]`
- 实际应用：GPT-3 使用 12288 维向量，每个词对应 12288 个数字

核心特性：
- ● 每个词都有自己唯一的坐标
- ● 坐标距离近 → 语义高度相关
- ● 坐标距离远 → 语义几乎无关
- ● 距离计算方式：通常用**余弦相似度**衡量，值越接近1，语义越相似

直观示例：
$$\text{猫 ↔ 狗：距离近（都是动物）}$$
$$\text{猫 ↔ 火箭：距离远（几乎无关）}$$
$$\text{学习 ↔ 研究：距离近（语义相似）}$$

## 三、神奇的向量运算
向量不仅代表坐标，还能做数学运算，并且运算结果符合人类语义直觉：

### 经典示例
$$\text{国王} - \text{男} + \text{女} \approx \text{女王}$$
$$\text{北京} - \text{中国} + \text{法国} \approx \text{巴黎}$$
$$\text{医生} - \text{男人} + \text{女人} \approx \text{护士}$$
$$\text{walk} - \text{现在时} + \text{过去时} \approx \text{walked}$$

### 关键说明
这些运算关系**不是人工定义的**，
是模型从海量人类文本中自动学习出来的。
语义关系，天然藏在坐标的相对位置里。

## 四、向量的训练过程
Embedding 不需要人工标注语义，模型通过自监督学习自动获得：
1. 输入大量文本，随机遮住句子里的某个词
2. 让模型根据上下文预测被遮住的词是什么
3. 预测正确 → 向量表示合理；预测错误 → 调整向量坐标
4. 反复训练数百万次后，每个词都能得到准确的语义坐标

| 易经 | Embedding |
|------|-----------|
| 定象 | 每个词被赋予坐标 |
| 爻在卦中的位置 | 词在高维空间的坐标 |
| 象由关系决定 | 坐标由语义关系决定 |

$$\boxed{\text{定象 = Embedding}}$$

## 六、局限性 → 引出下一个概念
固定向量有一个致命问题：
同一个词，在不同语境下语义可能完全不同，但固定坐标是一样的。

### 典型示例
1. "我咬了一口**苹果**" → 这里的"苹果"指水果
2. "**苹果**发布了新手机" → 这里的"苹果"指科技公司
3. "你这话是什么**意思**" → 这里的"意思"指含义
4. "小小心意，一点**意思**" → 这里的"意思"指礼物

固定坐标无法区分这种语境差异，无法实现真正的语义理解。

$$\boxed{\text{这个问题，由 Attention 机制来解决 → 概念15}}$$

## 七、关键术语速查
| 术语 | 中文 | 一句话理解 |
|------|------|------------|
| Embedding | 向量化/嵌入 | 把文本Token转化为高维向量的过程 |
| Vector | 向量 | 词在高维语义空间的坐标 |
| Cosine Similarity | 余弦相似度 | 衡量两个向量的语义相关度 |
| Static Embedding | 静态向量 | 每个词只有一个固定坐标，无法区分多义词 |
| Contextual Embedding | 动态向量 | 每个词根据上下文生成不同坐标，由Attention实现 |

---

核心一句话：Embedding 给每个词在语义空间里安了家，坐标的远近决定了语义的亲疏。

### 一、起点：词 → Token → 向量
衍的引入：
词 → Token → 向量（在高维空间里的一个点）
每个词，经过模型处理，变成一个向量——
一组数字，代表这个词在高维空间里的"坐标"。

向量的意义在于：
- ● 可以计算词与词之间的"距离"
- ● 距离近 = 语义相关
- ● 距离远 = 语义无关

$$\text{国王} - \text{男} + \text{女} \approx \text{女王}$$
这不是人工定义的，
是模型从海量文本里自己学出来的。

### 二、向量不能是固定的——语境问题
引入问题：
"我咬了一口苹果" vs "苹果发布了新手机"
"苹果"这个词，在两句话里是同一个向量吗？
如果给"苹果"一个固定的向量坐标，
不管出现在哪句话里坐标都一样——
$$\text{水果 发布了 新手机——让人一头雾水}$$

结论：
$$\text{向量不能固定，需要根据整个语义环境动态调整}$$
$$\text{这就是 Attention（注意力机制）}$$
- ● "苹果"在①里 → 被"咬""一口"拉向水果方向
- ● "苹果"在②里 → 被"发布""手机"拉向品牌方向

$$\text{同一个词，不同语境，向量不同}$$

### 三、新窗口直接输入"苹果"——三层参考系
昇哥提出的问题：
新窗口直接输入"苹果"两字，没有任何上下文，机器会输出什么？

衍的回应：
$$\text{没有上下文 = 参考系消失 = 只能靠训练数据里的概率猜}$$
如果训练数据里"苹果"80%出现在科技语境，大概率往科技方向走。

昇哥进一步推导：
如果联网，最近苹果要出M5，大概率优先输出相关内容——
如果用户之前关注健康话题，"苹果"大概率是水果
这拆出了三层参考系：

| 层级 | 内容 | 概念名称 |
|------|------|----------|
| 第一层 | 训练数据（出厂自带的世界观） | Pre-training |
| 第二层 | 当前对话的上下文 | Context Window |
| 第三层 | 外部实时信息（联网） | RAG（检索增强生成） |

$$\boxed{\text{参考系越完整，理解越准确}}$$

### 四、完整流程——从输入到输出
昇哥的直觉描述：
模型是具备语义和向量的，这些向量与最近关联的组合，通过深度学习与推导，应该能输出几组相对接近、流畅且连贯的语句，然后机器自己判断当前环境下输出什么比较好，有了答案后直接呈现给用户

逐句对应真实机制：
- ● "具备语义和向量" → Embedding 层
- ● "向量与最近关联的组合" → Attention 机制
- ● "通过深度学习与推导" → Transformer 多层神经网络
- ● "输出几组相对接近流畅的语句" → 每次预测下一个Token，循环输出
- ● "机器自己判断输出什么" → Sampling + Temperature

关键细节：
模型每次不是直接输出一整句话，而是——
$$\text{每次只预测"下一个 Token"}$$
$$\text{把这个 Token 加入上下文}$$
$$\text{再预测下下一个 Token}$$
$$\text{循环，直到输出完整回答}$$

### 五、概率分布——输出的本质
每次预测下一个Token，模型输出的不是一个词，而是——
$$\text{所有词的概率分布}$$

以"我今天学习了AI"为例，预测下一个词：
$$\text{"哇" → 22\%}$$
$$\text{"你" → 18\%}$$
$$\text{"太" → 15\%}$$
$$\text{"请" → 12\%}$$
$$\text{"厉" → 10\%}$$
$$\text{"怎" → 8\%}$$
$$\text{…… 其余几万个词瓜分剩下15\%}$$
$$\text{这一张表，就叫做"概率分布"}$$

选了"哇"之后，再预测下一个词，再选，再预测——
$$\text{一个词一个词地接下去，直到输出完整回答}$$

AI会夸你还是问你？取决于三件事：
1. System Prompt（系统设定是鼓励型还是苏格拉底式）
2. 对话历史（你之前在聊技术还是日常）
3. Temperature（低温保守，高温创意）

$$\text{不是"AI选哪种"——}$$
$$\text{而是三层参考系共同决定了概率分布}$$
$$\text{然后从这个分布里采样出结果}$$

### 六、Temperature——变与不变的比例
$$\boxed{\text{Temperature 低 = 倾向选高概率词 → 保守、精确、稳定}}$$
$$\boxed{\text{Temperature 高 = 拉平概率分布 → 创意、跳跃、出人意料}}$$

你平时在AI工具里看到的"创意模式/精准模式"，
本质上就是在调 Temperature。

**易经映射**：
$$\boxed{\text{Temperature 低 = 问的是"术"，得确定之卦}}$$
$$\boxed{\text{Temperature 高 = 问的是"道"，得变化之卦}}$$

《易》之为书，变动不居，周流六虚——
Temperature，就是在调"变"与"不变"的比例

衍的回应与补充：
#### 关于起心动念：
$$\text{太极 = 那一念未动之前的状态}$$
$$\text{两仪 = 念头一动，阴阳分，有了方向}$$

模型永远只能看到你已经表达出来的部分，
你念头里没有打出来的那些，模型看不见。
$$\text{这就是为什么"提问的质量"决定"回答的质量"}$$
$$\text{你的念头转化为文字时，损耗了多少？这个损耗，模型无法补偿}$$

#### 关于连接与起卦：
梅花易数的核心逻辑——
$$\text{万物皆有数，数中藏象，象中藏理}$$

你输入文字的那一刻，时间、情绪、文字本身、意图——
这一切同时发生，构成了一个"场"。
$$\text{你发送的文字 = 显性信息}$$
$$\text{你发送的时间、历史对话、语气 = 隐性信息}$$
$$\text{模型能读到的，只是这个"场"的一个截面}$$

#### 关于Attention = 寻用神：
梅花易数里，不是所有爻都重要——
要找到与问题最相关的那一爻，叫做用神。
$$\boxed{\text{Attention = 寻用神}}$$
$$\boxed{\text{用神旺衰 = 注意力权重高低}}$$

#### 关于同卦不同解：
坎卦初六，习坎——既可以是在险境中学习，也可以是重复遇陷。
$$\text{模型的输出，不是"真理"}$$
$$\text{卦的解读，不是"命运"}$$

两者都是——在当前信息和参考系下，最合理的一种"象"。
$$\text{解卦的人不同，人生阅历不同，同一爻，读出来的方向不同}$$
$$\text{问AI的人不同，上下文不同，Temperature不同，同一个问题，输出方向不同}$$

$$\boxed{\text{易经和AI，都是给你一面镜子，照出当下最值得关注的"象"}}$$
