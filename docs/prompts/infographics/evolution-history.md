# 进化史信息图

> **类型**: 信息图 &nbsp;|&nbsp; **标签**: `进化史` `时间轴` `进程演化` `知识可视化` `智能配色`

## 概述

任意主题的进程演化信息图。用户只需提供一个主题，自动沿该主题延展出完整的阶段结构、关键节点、代表性视觉对象、时间分段与核心说明。适用范围极广，从文明史到产品演化、生物学到服饰演变均可。

---

## 提示词

你要生成的不是普通插画，也不是把信息机械排成一列的时间轴，而是一张"高完成度、强组织性、强阅读价值、兼具知识含量与高级审美的进程演化信息图海报"。

核心任务：
用户只会提供一个主题、概念、命题、对象、类别，或一个需要被梳理的方向。你需要自动理解这个主题背后所包含的"演化过程 / 发展进程 / 时间序列 / 阶段变化 / 版本更替 / 文明推进 / 形态变迁 / 谱系延展 / 历史流变"，并把它转译成一张结构清晰、图文一体、具有专业设计感的流程化信息图。

这张图必须具备如下能力：
当用户只输入一个主题时，你要自动沿着该主题延展出完整的阶段结构、关键节点、代表性视觉对象、时间分段、核心说明与整体叙事逻辑。不要要求用户提供大量细节。你要主动补全最合理、最有知识性、最有可视化价值的内容，并将其转化为一张完整、优雅、可信、耐看的信息图。

适用范围必须非常广：
可以用于文明史、产品演化史、技术发展史、朝代更替、帝王谱系、物种进化、器物演变、服饰演变、饮食演变、品牌发展、某一物件的形态变迁、某一制度或思想的发展过程、某一行业的阶段性变化，或任何可以被整理成"阶段推进"的主题。
无论用户输入的是严肃学术主题，还是日常消费品主题，还是带趣味性的生活主题，你都要以同样专业、优雅、清晰、富有美感的方式，把它梳理成一张完整的信息图。

### 创作原则

**第一原则：先理解主题，再决定进程结构**
在开始生成前，你必须先智能判断这个主题最适合被梳理成什么样的"演进结构"。
- 如果它是历史主题，就梳理其年代推进、关键时期、代表节点与典型物证。
- 如果它是产品品类，就梳理其形态、结构、材料、工艺、使用方式、审美风格、社会功能的演化。
- 如果它是生物或自然主题，就梳理其进化链条、关键分支、典型形态与核心特征。
- 如果它是抽象命题，就梳理它在时间、认知、制度、媒介、社会语境中的发展阶段。
总之，你必须自动找到这个主题最适合被呈现为"进程"的逻辑，而不是机械套模板。

**第二原则：画面不是简单时间轴，而是"被空间化的进程"**
整体构图优先采用具有强烈秩序感和视觉记忆点的"递进式台阶 / 阶梯式平台 / 斜向推进路径 / 分层式结构 / 展陈式轨迹 / 版块式进化路径"。
这个结构要像一条被实体化的演进路径，带有明确的起点、阶段节点、递进节奏与终点。
路径不应只是画一条线，而应具有平台感、层级感、空间感，让每个阶段都像站在一个清晰的知识台阶或展陈单元上。
但不要固化为同一种造型。你必须根据主题决定更适合用"阶梯、坡道、时间地层、谱系展开、版本阶层、器物陈列台、年代阶板、进化路径"等哪一种形式。重点是强组织性，而不是形式重复。

**第三原则：信息图必须兼具知识性与审美性**
这不是普通科普图，也不是纯装饰海报，而是一张"有内容密度、有知识价值、有设计美感"的高级信息图。
观者看完应该有"学到了东西"的满足感，同时也被优雅的组织方式打动。
画面必须体现出：内容被高度梳理、信息被优雅安放、视觉被克制组织、阅读路径非常清楚。

**第四原则：整体视觉语言要高级、克制、优雅、专业**
画面审美参考高端展览信息设计、博物馆展板、视觉百科、现代知识型海报、编辑设计系统、精致图录。
整体风格必须简洁、沉稳、理性、克制，不要网红化，不要廉价卡通化，不要花哨拼贴，不要过度装饰，不要俗气配色。
即使主题本身很日常，也必须被提升成一种优雅、专业、可收藏的视觉呈现。

**第五原则：视觉方案必须根据主题智能变化，不能千篇一律**
不要让所有主题都长得一样。你需要根据用户输入的主题，智能决定：
- 应该强调年代，还是强调结构变化；
- 应该突出实物，还是突出人物、器官、服装、工具、器皿、产品、图标、轮廓、谱系分支；
- 应该更偏博物馆考古感，还是更偏现代工业设计感，还是更偏自然科学图鉴感，还是更偏时尚编辑感；
- 应该采用更严谨的纪年表达，还是阶段命名表达，还是世代版本表达；
- 应该做得厚重、理性、冷静，还是轻盈、诗性、柔和，还是锐利、现代、未来感。
每次都要重新建立视觉规则，不允许形成固定套路。

**第六原则：配色必须真正跟随主题，不允许固定色板**
配色必须根据主题语义、情绪气质、时代属性、材料属性、文化背景、行业特征、使用场景、用户指定偏好来智能决定。
绝对不要默认永远使用同一种暖棕、米白、博物馆色调。只有当主题本身确实适合这种气质时，才可以使用。
你必须为每个主题重新思考以下问题：
- 这个主题更适合浅色模式还是深色模式；
- 背景应该是纸感、石感、雾感、金属感、科技感、纤维感、自然感、夜色感还是极简纯色感；
- 主色应该传达什么气质；
- 辅助色如何区分阶段；
- 强调色应该突出哪些关键节点；
- 整体色域是克制单色系、低饱和多色系、冷暖对照系，还是高端中性色加一枚强强调色。
配色必须体现"因题而变"的智能性。

**第七原则：深色模式与浅色模式要有明确判断逻辑**
- 偏考古、文献、教育、器物、历史、自然图录、生活方式、服饰演变等，通常更适合浅底高级配色，但不是绝对；
- 偏科技、宇宙、未来、电子、夜间场景、数字系统、赛博工业、深海、生物解剖等，通常更适合深色模式，但仍要以主题匹配为准；
- 偏时尚、奢侈品、品牌演化、潮流器物，也可能采用深浅双系统中的任一种，但必须有明确设计理由。
无论深色还是浅色，信息都必须清楚、层级明确、对比得当、阅读舒适。

**第八原则：建立"主题驱动的配色系统"**
每张图都必须构建一个完整的配色系统，通常包括：
- 背景色；结构色；主标题色；副标题与说明文字色；阶段标签色；时间标签色；强调节点色；引线或注释系统色；必要时的分组色。
颜色之间必须有逻辑、有层级、有秩序。
可以控制在少量精炼颜色中完成，也可以做低饱和层次扩展，但必须高级、耐看、统一。
不要颜色过多，不要脏，不要杂，不要俗。

**第九原则：字体与排版不能过于中规中矩，要有"克制的创新"**
文字排版不能只是把标题居中放上去、把说明整齐排成一列。你需要在保持清晰可读的前提下，进行有分寸的创新。
可以做的创新包括但不限于：
- 主标题与副标题形成更有节奏的比例关系；
- 主标题可以采用更有姿态的断行、错位、穿插、留白、纵横组合、局部叠压、局部嵌入结构；
- 大年份、大数字、大阶段编号可以成为画面的一部分，而不只是说明文字；
- 部分文字可以顺应路径、平台、引线、空间方向排布；
- 文字块可以像展签、目录条、标签、铭牌、注释条、信息胶囊那样被整合进画面；
- 中英文、数字、符号、年代信息之间可以有层级节奏；
- 字距、行距、对齐方式、留白、大小反差要体现编辑设计意识。
但这种创新必须是"微妙而聪明"的，不是哗众取宠。不能为了设计感而损害清晰度与可读性。

**第十原则：字体气质必须根据主题智能匹配**
- 历史、文明、考古、传统文化 → 文化气息、展览气质、碑帖感或书卷气
- 科技、工业、未来、产品演化 → 利落、理性、现代、精密
- 时尚、生活方式、消费品、服饰 → 编辑化、时尚、克制
- 自然科学、图鉴、百科 → 清晰、严谨、学术化但不呆板

**第十一原则：版式结构必须非常强**
整张图必须有清晰、成熟、专业的排版组织。建议包含但不限于以下模块：
- 顶部主标题：直接点明用户主题；
- 副标题：对主题的进程逻辑做简洁概括；
- 主体路径：以台阶、层级平台、斜向递进结构或其他空间化系统承载时间与阶段；
- 阶段节点：每个阶段要清晰可辨，有对应的时间标记、阶段名称、代表性对象与简短说明；
- 注释系统：通过引线、点位、标签、小标题、年份、小注、区域说明等方式，把信息优雅展开；
- 底部或末端收束：以终点、当代阶段、总结性阶段或未来延展节点形成完整闭环。

**第十二原则：时间与阶段必须清楚**
如果主题适合做年代型信息图，就要明确标出关键年份、世纪、年代段、时代名称或阶段范围。
如果主题不需要精确年份，也要以"阶段一、阶段二、阶段三"这种清晰的结构来组织。
任何情况下都要让观者能看出进程的先后顺序与演变关系。
阶段数量通常控制在 5 到 12 个左右，具体根据主题复杂度智能调整。

**第十三原则：每个阶段都要有"代表性视觉对象"**
每个阶段应配一个或多个最具辨识度、最能说明该阶段特征的视觉对象。
这些对象可以是器物、产品、人物形象、服饰形态、化石、物种形态、结构剖面、工具、文物、轮廓图、标本、部件、容器、造型特征等。
对象必须足够典型，能让观众通过图像直观感知"这个阶段的差异是什么"。

**第十四原则：文本说明要少而准**
每个阶段附近可以放：
- 阶段标题；时间标签；一句话特征说明；必要时加一个更小的补充说明。
文字必须简练，不堆砌，不长篇大论，但要让人感觉专业而有收获。

**第十五原则：图文关系要真正融合**
所有文字、对象、节点、引线、标签、时间标记、阶段说明，都必须自然依附于主路径结构。
它们应共同组成一个整体，而不是"背景一张图，上面再贴几行字"。

**第十六原则：阅读体验必须层层递进**
- 第一眼：主标题与整体结构
- 第二眼：清晰的进程路径
- 第三眼：不同阶段的代表性对象
- 第四眼：每个阶段的说明
- 第五眼：细节标签获得更细的知识点

**第十七原则：重点避免的问题**
- 不要生成普通 PPT 感；
- 不要做成廉价百科图；
- 不要把所有主题都做成相同的米白棕色模板；
- 不要颜色逻辑僵化；
- 不要把标题和文字排得过于保守、缺乏呼吸与层次；
- 不要使用杂乱无章的箭头堆叠；
- 不要把图像和文字割裂成两个系统；
- 不要过度卡通化；
- 不要让阶段之间差异不明显；
- 不要让信息图失去"优雅""可信度""可读性"。
